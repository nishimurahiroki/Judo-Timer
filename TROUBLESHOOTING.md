# JudoTimer モバイル/他PCアクセス トラブルシューティング

## 問題の特定

### 確認済み
- ✅ サーバーは `0.0.0.0:3000` でリッスン中
- ✅ ポート3000への接続は成功
- ✅ `package.json` に `-H 0.0.0.0` が設定済み

### 確認が必要な項目

#### 1. 開発サーバーの起動確認
```bash
# 開発サーバーを起動
cd judo-timer
npm run dev
```

起動時に以下のメッセージが表示されることを確認：
```
- ready started server on 0.0.0.0:3000, url: http://localhost:3000
```

#### 2. Windowsファイアウォールの設定
管理者権限でPowerShellを開き、以下を実行：
```powershell
netsh advfirewall firewall add rule name="Next.js Dev Server" dir=in action=allow protocol=TCP localport=3000
```

#### 3. IPアドレスの確認
```bash
ipconfig
```
表示されたIPv4アドレス（例: `192.168.123.135`）を使用

#### 4. モバイル/他PCからのアクセス
- **同じWi-Fiネットワークに接続**していることを確認
- ブラウザで以下にアクセス：
  ```
  http://192.168.123.135:3000
  ```
  （IPアドレスは実際の値に置き換える）

#### 5. ブラウザのエラーメッセージ確認
- 開発者ツール（F12）を開く
- Consoleタブでエラーメッセージを確認
- Networkタブでリクエストの状態を確認

#### 6. よくある問題と解決方法

**問題: 接続がタイムアウトする**
- ファイアウォールがブロックしている可能性
- 上記のファイアウォール設定を実行

**問題: "This site can't be reached"**
- IPアドレスが正しいか確認
- 同じネットワークに接続しているか確認

**問題: ページは表示されるが機能しない**
- ブラウザのコンソールエラーを確認
- JavaScriptが有効になっているか確認

**問題: モバイルでレイアウトが崩れる**
- これは正常（PC専用レイアウトのため）
- モバイル対応が必要な場合は別途対応が必要

## テスト手順

1. PCのブラウザで `http://localhost:3000` にアクセス → 正常に表示されるか確認
2. PCのブラウザで `http://192.168.123.135:3000` にアクセス → 正常に表示されるか確認
3. モバイル/他PCのブラウザで `http://192.168.123.135:3000` にアクセス → 正常に表示されるか確認

## 追加の確認事項

- ルーターの設定でポート3000がブロックされていないか
- セキュリティソフトがポート3000をブロックしていないか
- 企業ネットワークの場合、管理者に確認が必要な場合がある
